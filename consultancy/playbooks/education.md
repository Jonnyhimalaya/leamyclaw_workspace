# Education Playbook
**Purpose:** Reusable playbook for courses, tutors, coaches, membership education businesses, and training companies.

---

## Why education businesses are a strong fit
These businesses usually have:
- recurring enquiries
- content release schedules
- membership/access management
- attendance/no-show issues
- repetitive student/parent questions
- content marketing demands
- founder overload

Excellent fit for agentic systems.

---

## Common Pain Points
- manual membership/course access updates
- inconsistent follow-up with leads/students/parents
- drip content handled manually
- lots of repetitive questions
- social content backlog
- no central view of enquiries, payments, access, and attendance
- important operational knowledge trapped in the founder's head

---

## Best Early Wins
### 1. Access / Membership Agent
- manages course access
- verifies membership state
- logs changes
- supports drip release workflows

### 2. Student / Parent Comms Agent
- drafts replies to common enquiries
- sends reminders
- handles onboarding/info flows

### 3. Content Release Agent
- weekly drip release
- artifact/report of what was released
- schedule-based publishing/access changes

### 4. Daily/Weekly Ops Briefing
- who paid
- who needs access
- who missed sessions
- what content goes live next
- open enquiries

### 5. Marketing / Content Agent
- social posts
- email campaigns
- course promo content
- testimonial / proof gathering ideas

---

## Suggested Agent Team
- Orchestrator
- Access/Membership Agent
- Student/Parent Comms Agent
- Content Release Agent
- Marketing Agent
- Scout / trend monitor

---

## Approval Model
### Safe early automation
- summaries
- access checks
- draft replies
- content scheduling recommendations

### Approve-then-act
- membership changes
- outbound parent/student emails
- page edits
- campaign changes

### Avoid early
- payment/refund handling
- pricing changes without approval
- deletion of student records

---

## Discovery Questions
- how do students buy/join?
- how is access granted?
- how is content released?
- what questions come up repeatedly?
- how are no-shows handled?
- how are payments tracked?
- what systems are used (WordPress, Teachable, Kajabi, WooCommerce, Stripe, etc.)?

---

## Strong Pilot Candidates
- membership/access workflow
- weekly content drip system
- enquiry triage + FAQ drafting
- weekly founder briefing
