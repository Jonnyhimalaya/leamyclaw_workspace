## Module 1 - Command centre

### Screenshot text
**Command centre**
The daily dashboard - one screen to run from

**What you actually need to see every morning**
- Occupancy vs this time last month + last year
- Top 3 tasks due today (pulled from your own task log)
- Reputation snapshot: review count, avg score, last 3 reviews
- Revenue vs forecast - single number, red/green
- One live social metric (reach or engagement, your pick)
- Upcoming local events in the next 14 days worth acting on

Tags:
- Zero setup daily
- At-a-glance only

### Interpretation
This is the operator home screen, not an analytics deep-dive.
It should answer: what matters today, what is off-track, and what needs action now.

### Required widgets
1. Occupancy comparator
2. Today task strip (top 3 only)
3. Reputation snapshot
4. Revenue vs forecast status chip
5. Single social pulse metric
6. Upcoming local events shortlist

### Product rule
Keep this page brutally compressed. No dense charts, no scrolling dashboard theatre, no second-order diagnostics on the first screen.

## Module 6 - AI task log

### Screenshot text
**AI task log**
What the AI did, what you approved, what it learned

**The accountability layer - so you always know what's running**
- Log of every AI-generated output: caption, response, brief, report
- Approval queue - nothing publishes without a human sign-off
- Prompt library - your best-performing prompts saved and reusable
- Feedback loop: mark outputs good/bad so the system improves over time
- Team activity feed - who approved what, when
- Monthly AI usage summary: time saved, tasks completed, error rate

Tags:
- Human-in-loop
- Audit trail

### Interpretation
This is governance, trust, and learning infrastructure for AI-assisted work. It makes the system legible and controllable rather than opaque.

### Required widgets / tools
1. AI output log across content, replies, briefs, reports
2. Human approval queue
3. Reusable prompt library
4. Good/bad feedback capture loop
5. Team activity feed / audit trail
6. Monthly AI usage summary

### Product rule
This module should make AI safe to rely on in a business context. The point is not showing off AI activity; it is preserving human control, traceability, reuse, and a compounding quality loop.
